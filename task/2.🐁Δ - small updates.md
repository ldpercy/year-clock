Keyboard shortcuts
------------------

Want to look at this next.
First want to fix the focus bug that's been in here for ages.

In turtle I've done this:
```js
	{
		query: 'textarea',
		type: 'keydown',
		listener: (event)=>event.stopPropagation()
	},
```
Which works okay, but maybe not the best idea here (or anywhere else) where I have more inputs.

Some form inputs also have their own default behaviours for selecting items based upon key presses, and I don't want to interrupt any of that.
So either cancel key event propagation on the form as a whole, or set the listener on the output/svg element????
Similar effect... is one better?
Might depend on unknown futures.
Actually one thing i should test in turtle is whether that stopPropagation actually kills alt/ctrl/meta events while inside the textarea.
Just tried, they're getting through okay with the current setup, phew.

Well I've decide to experimentally go for the stopPropagation technique as so:
```js
	{
		query: 'select,input',
		type: 'keydown',
		listener: (event)=>event.stopPropagation()
	},
```
Seems okay so far, and doesn't kill the whole form, just those inputs.
Still side-eyeing it a bit though - reckon there's a more elegant soln somewhere.

Have also change to kbd fn map and added synonyms for day + and -.
