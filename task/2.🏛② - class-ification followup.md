DayRange and Period Collections
-------------------------------

DayRange is a tentative class for storing day sequences.
It's meant to replace `getPeriodDayArray`, but I've only partially converted it so far.

In general though I need much better ways of doing these sort of things.
* contiguous day arrays for month and year
* arbitrary day collections for day-events
* contiguous period arrays for weeks, months, seasons etc
* arbitrary period collections for other kinds of notable time periods (terms, semesters, holidays etc)

So wondering whether they should all be the same type, or collections of the same type or something like that.


There are some features that days and time periods should have in common, and some less so.
Same with collections.
Try to get an organised picture and build from there.

* Whether an array/collection is contiguous or not shouldn't really make all that much difference after it's constructed i think. Keep an eye on, but probably unimportant.
* Days and Dates are probably different kinds of things if that helps. A date is like a coordinate system for time. Different coordinates could perhaps be used, such as unix time or various kinds of epoch times.
* Days and periods will both end up with Angular Ranges a lot of the time
* Pure *moments* in time would theoretically be represented by plain Angles
